# 로컬 작업 규칙

## 1. 문서 언어
- 모든 마크다운 문서는 기본값을 한글로 유지한다.
- API 경로, 환경 변수, 파일 경로, 제품 고유명이 아니면 사용자 노출 문구는 한글을 우선한다.

## 2. 인코딩
- 모든 문서는 `UTF-8`, `BOM 없음`, `LF` 기준으로 저장한다.
- 한글 깨짐을 막기 위해 문서 저장 후 `U+FFFD`와 연속 물음표 형태의 깨진 문자열 패턴을 확인한다.

## 3. 작업 로그
- 의미 있는 변경을 했으면 `codex.md`에 작업 내역을 기록한다.
- 실행 방식, 웹 구조, 공개 접속 구조가 바뀌면 `README.md`도 함께 갱신한다.
