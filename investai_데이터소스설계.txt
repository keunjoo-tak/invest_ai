주식 자동매매(신호+알림형) 데이터 원천 소스 설계서
범위
초기: 한국주식 (KOSPI/KOSDAQ 중심)
확장: 미국주식
전략: 멀티팩터 + 텍스트 보조형 (뉴스/공시/매크로/트윗)
실시간성: 수분 내 대응
실행범위: 신호 생성 + 알림 (카카오톡/SMS/텔레그램), 주문은 사람
1) 전체 원칙 (먼저 고정해야 할 운영 기준)
1.1 소스 계층 구조 (권장)
각 데이터 유형은 반드시 아래 3계층으로 설계합니다.
Tier 1 (공식/원천): 거래소, 감독기관, 정부통계기관, 기업 IR, 플랫폼 공식 API
Tier 2 (공식 집계/보조): 검색 API, 경제 데이터 허브(FRED 등), 브로커 API
Tier 3 (상용 벤더/대체): 유료 뉴스/시세/리서치 API, 스크래핑 대체
이 구조를 쓰면 향후 API 장애/요금 인상/약관 변경 시 교체가 쉽습니다.
1.2 수집 방식 원칙
정형 데이터(시세/지표/공시 메타): API 우선
문서형 데이터(공시 원문/IR자료/PDF): 원문 다운로드 + 파싱 파이프라인
검색형 데이터(뉴스/트윗): API + 검색 질의 규칙 + 중복제거
웹수집(스크래핑): 최후수단 (robots.txt/약관/법적 검토 필수)
1.3 시간 기준 표준화
저장 시각: collected_at_utc
이벤트 시각: event_time_local, event_time_utc
시장 기준:
KR: Asia/Seoul
US: America/New_York
모든 이벤트는 최종적으로 UTC 정규화 + 로컬시장시각 보존 권장
1.4 식별자 표준화 (매우 중요)
한국: 종목코드(6자리), ISIN, 시장구분(KOSPI/KOSDAQ/KONEX)
미국: Ticker, CIK, FIGI/CUSIP(가능하면)
내부 공통키: instrument_id
매핑 테이블 별도 운영 (티커 변경/합병/상장폐지 대응)
2) 데이터 유형별 원천 소스 상세 정의 (모든 케이스)

A-2. 미국주식 가격/거래량 (확장 단계)
[Primary 후보] 상용 시장데이터 API (Polygon /Finnhub 등)
Polygon 공식 문서 도메인에서 주식 관련 REST 엔드포인트(예: related tickers, ticker events 등) 확인 가능. 
Finnhub는 공식 API 문서에서 실시간 주가/펀더멘털/경제 데이터 등을 제공한다고 안내합니다.

실운영 전: 체결/호가/분봉 커버리지, 요금제, 지연 여부, 재배포 라이선스 비교
장기: 2개 벤더 병행(장애 대비)
대안(개발/백테스트용)
Alpha Vantage (저비용/간편하지만 호출량·지연·상용 제약 검토 필요)
공식 문서와 요금 안내 존재.

B. 뉴스 데이터 원천 (뉴스검색/텍스트 신호 핵심)
사용자 요구의 “웹서치(뉴스검색)” 부분.여기서는 “검색 API → 기사 URL 확보 → 본문 수집(허용 범위) → LLM 요약/감성/이벤트 태깅” 구조를 권장.
B-1. 한국 뉴스 (초기 핵심)
[Primary] 네이버 검색 API - 뉴스 검색
네이버 개발자센터의 Search API 뉴스 검색은 JSON/XML 반환, 검색어/정렬/조건 기반 조회가 가능하며, 검색 API 하루 호출 한도(25,000회)가 안내되어 있습니다. 
수집 데이터
제목

링크(URL)
언론사(가능 범위)
발행시간/정렬정보
검색어/질의 로그
수집 방식
REST 호출 (q=종목명 OR 회사명 OR 티커)
종목별 동의어 사전 사용
예: 삼성전자, 005930, Samsung Electronics
수집 후 URL 기준 dedup
기사 본문은 링크 원문 페이지에서 별도 수집(약관 검토)
운영 주의점
검색 결과 중심이므로 원문 전문(full text)이 항상 안정적으로 제공되지 않음
같은 기사 재송고/복제 기사 많음 → 클러스터링 필요
포털 정책/노출 로직 변화 영향 받음
[Secondary] Google Custom Search JSON API (사이트 제한 검색)
Google Custom Search JSON API는 Programmable Search Engine 기반으로 REST 검색 결과를 JSON으로 제공하며, 기존 고객 전환 안내(2027) 내용이 있습니다. 
용도
특정 도메인(예: 주요 경제지/기업 IR 도메인) site-restricted 검색
한국/영문 혼합 검색
뉴스 API 보강용
장점
도메인 지정 검색으로 정밀도 개선
웹문서/IR 페이지 탐색에 유용
[Fallback / 보조] 뉴스 전문 API 벤더 (NewsAPI 등)
NewsAPI는 전세계 뉴스 검색/헤드라인 API 문서를 제공하며 /everything, /top-headlines 등 엔드포인트가 있습니다. 
용도
글로벌 기사 검색
한국 외 확장 시 빠른 도입
헤드라인 수집 및 이벤트 탐지 보강
주의
한국 로컬 경제지 커버리지/지연/라이선스는 별도 검증 필요
상용 사용 약관 검토 필수
[보조 대안] GDELT (글로벌 뉴스/이벤트)
GDELT는 전세계 뉴스/이벤트 데이터셋과 API(예: DOC API, GKG 등) 기반 접근이 가능하며, 15분 단위 업데이트 관련 안내가 존재합니다. 
용도
매크로/지정학/국제 이벤트 감지
개별 종목보다 “시장 전체 위험 이벤트” 탐지
다국어 뉴스 흐름 보조 시그널
주의
개별 한국종목 정밀 매핑에는 후처리 필요
기사 본문보다 이벤트/문맥 신호 용도로 적합

D. 공식 경제지표 발표 데이터 (PPI/CPI 등) 원천

이 영역은 반드시 **“공표일정(Calendar)”**과 **“실제값(Result)”**를 분리 저장해야 합니다.

한국은행 ECOS는 Open API 서비스를 공식 제공하며 경제통계 활용용 API입니다.

KOSIS 공유서비스는 국가통계통합DB 접근 API를 제공하며 JSON/SDMX/XML/XLS 등 형식을 지원한다고 안내합니다.

D-1-3. 통계청 소비자물가 공표일정/보도자료 (Calendar + Release)
통계청(관련 페이지)에서 소비자물가동향 공표일정과 보도자료 목록이 공개됩니다. 2026년 공표일정 예시와 보도자료 목록 확인 가능.

CPI/PPI 개별 발표 페이지 및 릴리즈 일정 페이지가 존재합니다.

D-2-2. BEA (GDP, PCE, Personal Income 등)
BEA는 개발자용 API 및 개발자 페이지를 제공하며 JSON/XML 접근이 가능합니다. 
BEA 릴리즈 스케줄은 JSON/ICS 등 머신리더블 형식 안내가 있습니다. 
용도
GDP, PCE, 개인소득/지출 등
매크로 이벤트 캘린더 + 실제값 저장
D-2-3. FRED API (매크로 통합 조회/백테스트 보조)
FRED API는 경제 데이터 및 ALFRED(과거 vintage 데이터 포함) 접근을 제공하며 API 키가 필요합니다. 
용도
다양한 매크로 지표를 통합된 API로 빠르게 수집
백테스트/리서치 단계에 매우 유용
ALFRED로 리비전(개정치) 분석 가능
주의
원천 발표기관(BLS/BEA 등) 대비 중계 레이어이므로, 실시간 발표 해석은 가능하면 원천 기관 우선

공통 원칙

Degraded mode: 텍스트 소스 장애 시 가격+기술신호만 운영
Freshness 표시: 알림에 “최근 뉴스 데이터 지연” 표시 가능
예시

14) 데이터 품질 관리(DQ) 체크포인트 — 소스별 운영 규칙
자동매매(알림형 포함)에서는 “수집 성공”보다 “잘못된 수집을 빨리 감지”하는 것이 더 중요합니다.
14.1 가격 데이터 DQ
분봉 누락률 체크 (장중 기대 bar 개수 대비)
고가 < 저가 여부
거래량 급증 이상치 (벤더 오류 가능)
EOD와 intraday close 불일치 검출
자동 조치
재수집
보조 소스로 검증
신호 계산 일시 보류 플래그
14.2 뉴스 데이터 DQ
동일 URL 반복
제목/본문 빈값 비율
발행시각 파싱 실패율
특정 도메인 본문 추출 실패율 상승 감지 (파서 깨짐)
자동 조치
파서 버전 fallback
도메인별 extractor 규칙 업데이트 큐 생성
14.3 공시 데이터 DQ
신규공시 감지 건수 급감 (API/파서 장애 가능)
OPENDART vs KIND 카운트 괴리 모니터링
정정공시 체인 연결 실패율
수치 추출 필드 null 비율
14.4 X 데이터 DQ
수집량 급감 (quota/인증/플랜 문제 가능)
같은 post 반복 ingest
계정별 last_seen 시간 역전/정체
14.5 매크로 데이터 DQ
calendar에 있는 발표가 actual로 안 들어오는 경우
실제값 숫자 파싱 실패
단위/기간(label) 파싱 오류
revised value 반영 누락
