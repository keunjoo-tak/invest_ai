# Stock Decision 리포트

## 1. 목적
`Stock Decision`은 특정 종목을 현재 시점에서 어떻게 봐야 하는지 설명하는 핵심 사용자 제품입니다.

핵심 질문은 다음과 같습니다.
- 지금 이 종목을 매수 관점으로 볼 수 있는가?
- 보유 유지가 맞는가?
- 아직 관찰이 우선인가?
- 비중 축소가 필요한가?

## 2. 호출 방식
- 엔드포인트: `GET /api/v1/stock-decision/{ticker_or_name}`
- 웹 화면: `조건 입력 후 실행` 버튼 클릭 -> 슬라이드 입력 패널 -> 실행

## 3. 입력
- `ticker_or_name`
- 선택적 `as_of_date`
- 선택적 `lookback_days`

## 4. 출력 리포트 구조
### 첫 영역
- 최종 결론
  - 예: `삼성전자는 현재 보유 관점이며, 상태는 상승 추세입니다.`

### 두 번째 영역
- 판단 점수
- 품질 점수
- 시그널 점수
- 문서 근거 수

### 세 번째 영역
- 기간별 판단 강도
  - 단기
  - 스윙
  - 중기
  - 시장
  - 섹터
  - 종목 고유
- 활용 데이터 요약
- 상승 요인
- 하락 요인
- 재무 근거
- 거시/정책 근거
- 최근 이벤트 타임라인
- 데이터 출처

## 5. 활용 데이터
- 시세: KIS
- 뉴스: NAVER News, NewsAPI
- 공시: OpenDART, KIND
- 재무제표: OpenDART 재무제표 API
- 거시/정책: BOK, KOSIS, 정책브리핑, 한국은행, 글로벌 거시 데이터

## 6. 처리 흐름
1. 공통 분석 엔진(`POST /api/v1/analyze/ticker` 내부 파이프라인)으로 시세/뉴스/공시/재무/거시를 수집
2. feature 생성
3. signal 계산
4. 사용자 제품 관점으로 결론과 설명을 재구성
5. 리포트 형태로 출력

## 7. 사용자 신뢰 포인트
- 결론만이 아니라 상승/하락 요인을 함께 제공
- 재무 요약과 거시/정책 요약을 별도로 표시
- 최근 이벤트 타임라인으로 왜 이런 결론이 나왔는지 설명
- 활용 데이터 출처를 명시
