# Market Regime 리포트

## 1. 목적
`Market Regime`는 사용자가 종목을 보기 전에 먼저 확인해야 하는 상위 시장 환경을 제공합니다.

핵심 질문은 다음과 같습니다.
- 오늘 시장은 공격적으로 볼 구간인가?
- 방어적으로 접근해야 하는가?
- 강한 섹터와 약한 섹터는 어디인가?

## 2. 호출 방식
- 엔드포인트: `GET /api/v1/market-regime/overview`
- 웹 화면: 서비스 카드에서 `리포트 실행` 버튼 클릭 시 즉시 호출

## 3. 입력
- 기본적으로 추가 입력 없음
- 선택적 `as_of_date`를 API 직접 호출 시 사용할 수 있음

## 4. 출력 리포트 구조
### 첫 영역
- 최종 결론
  - 예: `현재 시장은 위험선호 국면입니다. 반도체 중심의 선별 접근이 유리한 구간입니다.`

### 두 번째 영역
- 체제 점수
- 강세 섹터 수
- 약세 섹터 수
- 대표 종목 수

### 세 번째 영역
- 활용 데이터 요약
- 전략 힌트
- 강세 섹터
- 약세 섹터
- 거시 근거
- 대표 종목
- 데이터 출처

## 5. 활용 데이터
- 가격 데이터: KIS 일봉
- 거시 데이터
  - BOK
  - KOSIS
  - OECD
  - FRED
  - BLS
  - BEA
  - Fiscal Data
  - World Bank
  - IMF
  - Eurostat
- 문서 데이터
  - 정책브리핑
  - 한국은행 자료
  - 글로벌/국제 거시 브리핑 배치 결과

## 6. 처리 흐름
1. 메모리 캐시 확인
2. `product_snapshot_cache`의 시장 체제 스냅샷 확인
3. 스냅샷이 없으면 실시간 수집으로 계산
4. 결과를 리포트 형태로 정리해 화면에 출력

## 7. 사용자 신뢰 포인트
- 단순 점수만 보여주지 않고, 강세/약세 섹터와 거시 근거를 함께 노출
- 응답 경로(`메모리 캐시`, `배치 스냅샷`, `실시간 수집`)를 표시
- 어떤 데이터가 반영됐는지 리포트 하단에 설명
