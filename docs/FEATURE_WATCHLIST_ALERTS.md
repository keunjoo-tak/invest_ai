# Watchlist Alerts 리포트

## 1. 목적
`Watchlist Alerts`는 즉시 대응이 필요한지 점검하고, 관찰 종목을 저장형 워치리스트로 관리하는 서비스입니다.

## 2. 호출 방식
### 즉시 점검
- 엔드포인트: `POST /api/v1/watchlist-alerts/check`
- 웹 화면: `즉시 점검` 버튼 클릭 -> 슬라이드 입력 패널 -> 실행

### 저장형 워치리스트
- 생성: `POST /api/v1/watchlist-alerts/subscriptions`
- 조회: `GET /api/v1/watchlist-alerts/subscriptions`
- 삭제: `DELETE /api/v1/watchlist-alerts/subscriptions/{ticker_or_name}`
- 웹 화면: `워치리스트 저장` 버튼 또는 카드 하단 목록 사용

## 3. 입력
### 즉시 점검
- `ticker_or_name`
- `notify`
- `force_send`

### 저장형 워치리스트
- `ticker_or_name`
- `notes`
- `channel`

## 4. 출력 리포트 구조
### 첫 영역
- 즉시 점검 필요 여부
  - 예: `삼성전자는 현재 즉시 대응보다 관찰 유지가 우선입니다.`

### 두 번째 영역
- 즉시 알림 여부
- 모니터링 상태
- 시그널 점수
- 리스크 플래그 수

### 세 번째 영역
- 즉시 확인할 트리거
- 리스크 플래그
- 관찰 촉매
- 활용 데이터 요약
- 알림 메시지 미리보기
- 데이터 출처

## 5. 활용 데이터
- 종목 시세
- 뉴스/공시
- 재무제표
- 정책/거시 데이터
- 알림 채널 설정

## 6. 처리 흐름
1. 현재 시점 데이터를 다시 수집 또는 캐시 조회
2. 시그널 점수와 리스크 플래그 확인
3. 알림 필요 여부 계산
4. 사용자에게 즉시 점검이 필요한지 리포트로 제공
5. 저장형 워치리스트는 DB에 별도 저장

## 7. 사용자 신뢰 포인트
- 즉시 대응 여부와 그 근거를 분리해 제공
- 알림 메시지 미리보기를 함께 노출
- 저장된 워치리스트 목록을 같은 화면에서 관리 가능
