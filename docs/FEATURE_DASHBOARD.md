# 대시보드 화면 구조

## 1. 목적
현재 웹 화면은 투자자가 한 화면에서 서비스 선택, 조건 입력, 결과 리포트 확인, 최근 실행 이력 확인을 수행할 수 있도록 구성되어 있습니다.

## 2. 레이아웃
### 2.1 좌측 컨트롤 레일
- 서비스 이동 버튼
- 현재 선택한 서비스의 입력 패널
- 최근 실행 이력
- 저장형 워치리스트 요약

### 2.2 우측 리포트 스테이지
- 선택한 서비스의 결과 리포트를 단일 영역에 출력
- 리포트는 카드형 섹션으로 구성
- 세부 섹션은 접기/펼치기 가능

### 2.3 보조 드로어
- 운영자 도구를 본문과 분리하여 우측 드로어로 제공
- 배치 실행, 소스 미리보기, 내부 점검 기능 포함

## 3. 상단 영역
- 서비스 제목 및 한 줄 소개
- 서버 상태 표시
- 현재 로그인 사용자 표시
- 새로고침 버튼
- 로그아웃 버튼

## 4. 최근 실행 이력
- 최대 10건을 보관
- 화면에는 약 3건 높이만 보이도록 스크롤 처리
- 클릭 시 해당 리포트를 다시 열람 가능

## 5. 모바일 대응
- `768px` 이하에서는 좌측 레일과 우측 리포트를 세로 배치
- `560px` 이하에서는 카드 여백과 버튼 크기를 조정
- 표와 로그 영역은 가로 스크롤 가능

## 6. 디자인 원칙
색상, 폰트, 카드 스타일은 유지하고 정보 구조만 조정하는 방향으로 설계되었습니다.
