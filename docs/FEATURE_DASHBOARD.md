# 대시보드 경험 문서

## 1. 개요
현재 웹 화면은 단순 API 테스트 화면이 아니라, 투자 판단 흐름에 맞춘 리포트형 대시보드입니다.

사용자 동선은 다음 순서를 전제로 설계했습니다.
1. `Market Regime`로 오늘 시장의 상위 맥락 확인
2. `Stock Decision`으로 개별 종목 판단 확인
3. `Action Planner`로 실제 행동 계획 정리
4. `Watchlist Alerts`로 관찰 종목 저장 및 점검

## 2. 화면 구조
### 상단 영역
- 서비스 개요와 현재 백엔드 상태 표시
- 사용자는 먼저 현재 서비스가 정상 동작 중인지 확인 가능

### 서비스 카드 영역
- 카드 1: `Market Regime`
  - 즉시 실행 버튼
- 카드 2: `Stock Decision`
  - 입력 패널 호출 버튼
- 카드 3: `Action Planner`
  - 입력 패널 호출 버튼
- 카드 4: `Watchlist Alerts`
  - 즉시 점검 버튼
  - 워치리스트 저장 버튼
  - 카드 하단에 저장형 워치리스트 목록 표시

### 운영자 도구 영역
- `Source Preview`
- `Batch Execution`

## 3. 입력 방식
입력이 필요한 서비스는 화면 중앙에 폼을 늘어놓지 않고, 오른쪽 슬라이드 패널에서 파라미터를 받습니다.

### 목적
- 메인 화면은 리포트 소비에 집중
- 입력이 필요한 순간에만 파라미터 패널 노출
- 증권사 앱처럼 결과 화면과 입력 화면을 분리

## 4. 결과 출력 방식
각 서비스 버튼 아래쪽에 결과가 리포트 형식으로 출력됩니다.

### 리포트 공통 원칙
- 첫 줄: 사용자가 바로 이해할 수 있는 최종 결론
- 두 번째 영역: 현재 응답이 어떤 경로에서 왔는지 표시
  - 메모리 캐시
  - 배치 스냅샷
  - 실시간 수집
  - 실시간 점검
- 그 다음 영역: 활용 데이터 요약
- 마지막 영역: 세부 근거, 타임라인, 데이터 출처

## 5. 신뢰를 위한 표시 요소
모든 주요 리포트에는 다음 정보를 명시합니다.
- 활용된 데이터 요약
- 핵심 점수 또는 상태
- 최근 이벤트/문서 타임라인
- 거시/정책/재무 근거
- 데이터 출처 설명

## 6. 현재 디자인 방향
- 금융 서비스형 카드 대시보드
- 서비스별 색상 구분
- 리포트 우선 화면 구성
- 운영자 기능은 별도 영역으로 분리
- 모바일과 데스크톱 모두 동작 가능한 반응형 구조
