“서비스 선택 화면”과 “리포트 열람 화면”이 같은 레이아웃 안에 섞여 있다는 점이 문제야.
화면 정보구조(IA)와 레이아웃 모드 전환 문제를 해결해줘.

기본적으로 컨텐츠 배치, 구조를 수정하는 것이지, 전반적인 웹 페이지의 색감이나 디자인, 폰트 등 디자인적 요소들은 현재 상태를 **반드시** 유지합니다.

1) 가장 중요한 원칙: “선택 화면”과 “결과 화면”을 분리
지금처럼 홈에서 바로 카드 아래에 결과를 넣지 말고, 서비스 실행 시 결과 전용 뷰로 전환
# 권장 구조
-홈 화면
4개 서비스 카드
각 서비스 목적 설명
빠른 실행 버튼
-결과 화면
선택한 서비스의 전체폭 리포트
좌측 또는 상단에 파라미터/조건 입력
우측 또는 하단에 리포트 본문
필요시 다른 서비스로 전환 가능

2) 가장 추천하는 레이아웃: “좌측 컨트롤 + 우측 리포트” 2단 구조
- 전체 레이아웃
상단: 글로벌 헤더
좌측(320~380px 고정폭): 서비스 선택 + 입력 조건 + 실행 히스토리
우측(가변폭, 핵심 영역): 리포트 본문 전용 영역

이 구조의 장점
-사용자는 입력과 결과를 분리해서 인지할 수 있습니다.
-결과 리포트가 전체 폭을 충분히 사용할 수 있습니다.
-서비스가 바뀌어도 레이아웃은 동일하므로 UX 일관성이 좋아집니다.

3) 리포트는 “스크롤 문서형”으로 봐야 함

4) 각 리포트는 “요약 → 펼침” 구조로 바꿔야 함
Level 1. 상단 Executive Summary : 맨 위에서 바로 보여줘야 하는 것만 노출
Level 2. 섹션 요약 카드 : 각 섹션을 한 줄 요약으로 접어둠
Level 3. 상세 내용

지금처럼 처음부터 전부 보여주지 말고,
처음엔 “판단에 꼭 필요한 최소 정보”만 보이게 해줘.

5) 결과 상단은 반드시 “판단 요약 바”로 고정
상단에는 sticky summary bar 고정

6) 리포트 본문은 “한 줄 폭”을 줄여야 함
현재 이미지를 보면 좁은 폭 안에 텍스트가 길게 접히면서 읽기 어려운 상태임.
그런데 반대로 전체폭을 무조건 다 쓰면 텍스트 줄 길이가 너무 길어져서 또 읽기 어려움.
그래서 리포트 본문은 아래와 같이 재구성

권장 폭
본문 읽기 영역 최대폭: 960~1200px
본문 내부 텍스트 컬럼 최대폭: 720~820px
수치 카드/보조 패널은 옆 컬럼으로 배치

즉, 전체 화면을 넓게 쓰되 텍스트 자체는 너무 넓게 퍼지지 않게 해야 합니다.

7) 리포트 내부는 “1열 문서 + 1열 보조 패널” 구조가 좋음
-메인 컬럼(약 70%)
결론
상세 해석
근거 문장
타임라인
반증 조건
-보조 컬럼(약 30%)
핵심 점수
요약 KPI
빠른 액션
리스크 플래그
관련 이벤트 일정

이렇게 하면 리포트는 문서처럼 읽히고, 오른쪽에는 언제든 참고할 수 있는 핵심 요약이 노출.
특히, Stock Decision, Action Planner에 잘 맞습니다.

8) 서비스별 화면 패턴을 통일하되, 리포트 구조는 다르게
지금 4개 서비스는 성격이 다릅니다.
따라서 카드 UI를 똑같이 유지할 필요는 없고, 상위 레이아웃만 통일하고 내부는 서비스별로 최적화한다.

-Market Regime
가장 적합한 패턴:
상단: 시장 체제 결론 + 점수 + 확신도
중단: 체제 분해 / breadth / 거시 압력
하단: 강세/약세 섹터, 핵심 동인, 반증 조건

이 서비스는 “시장 맵” 형태가 중요합니다.
즉 텍스트보다:
-체제 score bars
-섹터 heatmap
-주요 동인 카드
-이벤트 캘린더
같은 시각화 비중을 높여줘.

-Stock Decision
가장 적합한 패턴:
상단: 최종 판단 + 투자 포인트 + 기간별 점수
좌측 본문: 상승/하락 근거, 타임라인, 재무/이벤트 해석
우측 보조패널: 핵심 지표, 리스크, 체크포인트

이 서비스는 문서형 분석 보고서가 가장 잘 맞습니다.

-Action Planner
가장 적합한 패턴:
상단: 지금 해야 할 행동
중단: 진입 구간 / 무효화 구간 / 목표 구간
하단: 시나리오별 대응

이건 문서형도 필요하지만 플로우차트/시나리오 카드 형태가 매우 중요합니다.

-Watchlist Alerts
가장 적합한 패턴:
리스트 중심
우측 상세 패널
알림 상태/트리거 중심

이 서비스는 긴 리포트보다 모니터링 콘솔형 UI가 더 적합합니다.

즉, 4개 서비스에 동일한 리포트 레이아웃을 강제하지 않는 것이 좋음.

9) 한 화면에 “여러 서비스 결과”를 동시에 보여주지 않는 게 좋음

지금은 2x2 레이아웃 때문에 다른 서비스 칸들이 비어 있는 상태에서 한 칸만 리포트가 길게 열리고 있습니다. 이건 시각적으로도 “화면 낭비”

더 나은 방식

서비스 실행 시:
-선택한 서비스만 메인으로 확대
-나머지 서비스는 좌측 탭 또는 상단 탭으로 축소
-필요하면 “이전 서비스 결과와 비교”만 보조 기능으로 제공

즉, 동시에 4개 서비스를 다 보게 하지 말고
지금 보고 있는 서비스 하나에 집중시키는 게 맞습니다.

10) 추천하는 최종 화면 구조
-상단 헤더
서비스명
시장/종목 검색
실행 상태
저장 / 공유 / 내보내기

-좌측 사이드바
4개 서비스 탭
서비스별 입력 패널
최근 실행 이력
워치리스트

-메인 영역
상단: 리포트 헤더 + 핵심 결론 바
본문:
1행: KPI 카드 4~6개
2행: 핵심 결론 / 근거
3행 이후: 상세 섹션 아코디언
우측 서브패널:
빠른 요약
리스크
예정 이벤트
관련 액션 버튼

11) 디자인 레벨에서 바로 바꿔야 할 것
즉시 수정 권장
-2x2 카드 안에서 결과 렌더링 금지
-결과는 별도 전체폭 영역에서 렌더링
-서비스 실행 시 선택 카드만 확장하거나 결과 화면으로 전환
-리포트 상단에 sticky summary bar 추가
-본문은 세로 문서형으로 재배치
-섹션별 접기/펼치기 적용
-긴 텍스트 영역은 최대 너비 제한
-강세/약세 섹터, 점수 분해는 카드 나열보다 시각화 강화
-Watchlist Alerts는 리포트형보다 리스트+상세형으로 변경
-운영 도구는 별도 관리자 드로어/모달로 분리
